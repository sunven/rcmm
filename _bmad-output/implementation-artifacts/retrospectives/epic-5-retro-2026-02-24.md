## 成功之处

### 1. MenuBarExtra window 样式迁移
- 从 menu 下拉迁移到 window 弹出
- 更好的用户体验，支持自定义布局
- PopoverContainerView 状态路由

### 2. ActivationPolicyManager 集中管理
- 消除散落在 rcmmApp.swift 和 AppState.swift 的激活策略代码
- 提供 showSettings() 和 hideToMenuBar() 方法
- 更清晰的职责分离

### 3. HealthStatusPanel 组件
- 三种状态：enabled（绿色）、unknown（黄色）、disabled（红色）
- 图标 + 颜色 + 文字三重传达（色盲友好）
- 完整的无障碍支持

### 4. SMAppService 开机自启
- 从系统读取真实状态（不依赖 UserDefaults）
- Toggle + onChange 模式，含错误回退
- 与引导流程复用同一登录项

### 5. #Preview 模式建立
- 所有新 View 都包含预览变体
- 多状态 + Light/Dark Mode 覆盖

---

## 挑战与改进

### 挑战

1. **MenuBarExtra 样式迁移**
   - 从 menu 到 window 是破坏性变更
   - 需要重新设计交互流程

2. **onChange 递归触发**
   - Toggle 赋值触发 onChange，onChange 中又赋值 Toggle
   - 解决方案：使用 isUpdating 守卫标志

3. **废弃 API 处理**
   - NSApp.activate(ignoringOtherApps:) 已废弃
   - 解决方案：使用 NSApp.activate()

### 技术发现

| 发现 | 影响 | 处理 |
|------|------|------|
| SMAppService.mainApp.status 是唯一状态源 | 不能缓存到 UserDefaults | 直接读取系统状态 |
| MenuBarExtra window 样式更灵活 | 支持自定义 View | 已迁移 |
| onChange 赋值会递归触发 | 需要 isUpdating 守卫 | 已实现 |
